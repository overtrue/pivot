## 💡 设计原则

### 🎯 原子化设计

每个组件都遵循原子化设计原则，可以独立使用，也可以组合构建复杂界面：

```tsx
// 原子组件组合
<div className="flex items-center gap-2">
  <MethodLabel method="GET" />
  <StatusCode code={200} />
  <RequiredBadge />
</div>

// 分子组件
<ParameterItem
  parameter={param}
  components={components}
/>

// 有机组件
<OperationDetail
  operation={operation}
  path="/users/{id}"
  method="GET"
  components={components}
/>
```

### 🔧 可组合性

组件设计注重可组合性，支持灵活的组合使用：

```tsx
// 布局组合
<OperationDetailedLayout
  spec={spec}
  selectedPath="/users"
  selectedMethod="GET"
  showCodegen={true}
  showTryPanel={true}
/>
```

### 🎨 一致的视觉语言

所有组件遵循统一的设计系统：

- **颜色系统**：基于语义化的颜色定义
- **间距系统**：8px 基础网格系统
- **字体系统**：层次化的字体大小和权重
- **圆角系统**：统一的圆角规范

### ♿ 无障碍设计

所有组件都遵循 WAI-ARIA 标准：

- **键盘导航**：完整的键盘操作支持

## 🤝 贡献指南

### 📋 开发标准

1. **组件命名**：使用 PascalCase
2. **文件命名**：使用 kebab-case
3. **Props 接口**：提供完整的 TypeScript 类型
4. **样式处理**：使用 Tailwind CSS + CVA
5. **文档要求**：每个组件都要有使用示例
